# helpdesk
プロジェクト「ヘルプデスクONE」企画書
作成者: 情報システム部

対象規模: 従業員 30,000名

プロジェクトコンセプト: 「AI-First, Human-Centric Support」

1. エグゼクティブ・サマリー
1.1 プロジェクトの目的
従業員30,000名規模のITサポートを安定稼働させるには、業界標準（1:200比率）で約150名の体制が本来必要とされます。本プロジェクトは、生成AI（LLM）と10名以内の少数精鋭チームを融合させた「ハイブリッド・サポートモデル」を構築することで、標準モデル比で約93%の人的リソース削減を実現しつつ、24時間365日の即時解決を目指します。

1.2 期待される成果
圧倒的コスト回避: 本来必要な150名体制に対し、年間約 8.4億円規模 の潜在的人件費を回避。
属人化の解消: 全知見をAIナレッジ基盤（RAG）へ統合し、スキルの平準化を達成。
DXの加速: 定型業務をAIへ移譲し、IT部門のリソースを戦略的業務へシフト。
2. 運用モデルの比較（標準 vs 本案）
比較項目	A. 業界標準モデル（理論値）	B. ヘルプデスクONE（採用案）
運用形態	人海戦術による電話・メール対応	AI主導・人間管理のハイブリッド
必要人員	約 150名 (1:200)	10名以内 (1:3000)
年間人件費	約 9.0億円 (600万円/名)	約 0.6億円 (600万円/名)
ナレッジ資産	個人の経験に依存（属人化）	社内DBに構造化され蓄積
拡張性	低い（増員が必須）	高い（AI処理能力の増強のみ）
判定	コスト・管理負荷により不採用	採用（唯一の現実解）
3. リソース算出の根拠
3.1 業界標準（150名）の根拠
ITサービスマネジメント（HDI/Gartner）の標準的な指標である「サポート・レシオ」に基づきます。
適正配置比率: ユーザー200名に対し、サポート担当1名。
計算式:
30,000名÷200=150名
稼働詳細: 月間3万件の問い合わせを、1件15分（AHT）で処理する場合、欠勤率やピークタイムを考慮すると、安定運用には150名が理論上の下限となります。
3.2 本プロジェクト（10名）の実現性
AIを単なる「補助」ではなく「一次対応の主体」に据えることで、構造を変革します。

効率化の要因	効果のメカニズム	削減寄与率
Tier 0 (AI自律解決)	頻出質問（パスワード、Wifi等）をAIが完結。	50%〜70%
AI Co-Pilot (回答支援)	AIが回答案を即時生成。調査・入力時間を大幅短縮。	20%
ナレッジ統合 (RAG)	属人化を排除し、誰でもベテラン並みの速度で回答可能。	10%
4. 「ヘルプデスクONE」の構造と解決フロー
10名のチームで3万人を支えるため、以下の「3層防衛ライン」を構築します。
Tier 0（AI自律解決）: AIがRAGを参照し、チャットで即答。
Tier 1（AI支援・有人確認）: AIが回答案と根拠を提示し、10名のオペレーターが承認・修正して送信。
Tier 2（エキスパート）: AI解決不能な難問を調査。解決策を即座にAIへ学習させ、同様の質問の再発を「自動化」で扑滅。
5. 重点リスク管理（主要課題への対応）
分類	リスク内容	具体的対策（緩和策）
品質	AIの誤回答	Co-Pilot運用: 最初は内部利用に限定。回答には社内ソースのリンクを必須化。
運用	オペレーターの負荷増	リスクベース監査: 低リスク案件はAIに任せ、人間は高リスク案件に集中。
ナレッジ	情報の劣化	ライフサイクル管理: ナレッジに有効期限を設け、期限切れ記事をAIが自動検知。
組織	スキル劣化・離職	教育・標準化: 定期的なローテーションを実施し、業務を徹底的に標準化。
6. ロードマップ（段階的導入計画）
Phase 1：基盤・データ整備（1-3ヶ月目）
既存マニュアルの断捨離。セキュリティ基準（PII保護）の策定。
Phase 2：内部試運転（4-6ヶ月目）
ユーザー非公開。 オペレーターがAI支援ツールを使い、精度を磨き込む。
Phase 3：限定公開・精度検証（7-9ヶ月目）
特定部署への先行リリース。AIの自信度が高い回答のみ自動返信を許可。
Phase 4：全社展開（10ヶ月目〜）
全社公開。電話窓口を原則廃止（予約/IVR化）し、チャット誘導を徹底。
7. 予算計画とROI
回避される人的コスト: 約 8.4億円 / 年（標準モデル150名との差分）
初期投資: 約 3,000万円（AI環境構築、ナレッジ整備）
年間運用費: 約 3,000万円（システム費、AIトークン代）
実質的な年間コストメリット: 約 8.1億円 / 年
算出式:
人件費削減額 (8.4億)−運用費 (0.3億)=8.1億円
8. プロジェクトの成否を握る「条件」
「AIは育てるもの」という認識: Day 1から完璧を求めない文化を醸成する。
ナレッジオーナーシップ: 情シスだけでなく、情報の発生源（人事、総務等）が責任を持つ。
業務導線の変革: 「電話からAIへ」の移行を経営課題としてトップダウンで推進する。
